# 求职简历建议汇总

`更新 26-01-27` \| `发布 24-10-30`

*[申明] 所有建议仅供参考。觉得适合自己的建议，就酌情参考后采纳。觉得和自身情况不大相符的建议，略过就好。*

[摘要] 本文对如何写求职简历给出一些建议，用于呈现必要信息，以提升获取 offer 的几率。本文综合了公开信息、大厂HR给的信息、优化简历工作中获得的信息，等。

## 整体建议

有如下整体建议，供参考。

✳️ **每个字都是为了“我很强！我很适合该岗位！”**

- 惜墨如金。不写不相关的，或不重要的。比如通常不写有驾照。
- 通常写满 1 页 A4 纸。不满 1 页，或者稀稀拉拉凑满 1 页，会被认为能力不行。
- 通常不超过 1 页。除非真的要写 2 页。26届入职阿里的同学的求职简历就是 2 页。

✳️ **诚信，不造假**

- 一旦被用人单位发现造假，则无望获得 offer。

✳️ **呈现最美的80%，不主动暴短**

- 比如只呈现若干核心课程满绩，但 GPA 3.0 就不写在简历上了。（GPA3.0只是举例，主要看专业或年级排名）
- 比如不主动呈现 `GPA 2.0，专业排名前 90%`。但要想好用人单位问 GPA 时怎么回答，不能造成脑子不聪明、学习不行的印象。

✳️ **重要的往前放**

- 排在前面，排在左边。符合从上到下、从左到右的阅读习惯。
- 比如 `国家一等奖学金`，就放在 [荣誉奖项] 那行的最左边。
- 比如有在大厂相关的实习经历，就把 [实习经历] 放到 [项目经历] 前面。

✳️ **排版优美，逻辑清晰**

- 我技术好，不整这些虚的。要整这些虚的！很多985、海归，不仅技术好，简历更精致。
- 求职简历逻辑清晰、优美精致，应是加分项。
- 用好看的字体，大小适中。既高级又便于阅读。
- 类似内容的字体要统一。比如项目1用宋体、18px，项目2用微软雅黑、24px，就显得不统一。
- 合适的行间距，以方便阅读。不要行与行紧紧挤在一起。
- 合适的字间距（如可以调的话），以方便阅读。
- 可加粗和（或）不同颜色，突出希望用人单位看到的信息。重点也不要过多，过多就没有重点。
- 排版四大原则：对比、重复、对齐、亲密。

✳️ **正确使用英文大小写**

- 我技术好，不整这些虚的。要整这些虚的！科研和工作都要严谨！
- 我是 MySQL。我不是 MYSQL、mysql、mySQL。
- 我是 Python。我不是 python、PYTHON。

✳️ **正确使用中文标点符号**

- 该逗号就逗号，该顿号就顿号。不要一逗到底。
- 更多详情请参考 [国标 GB/T 15834-2011 标点符号用法 🔗](https://openstd.samr.gov.cn/bzgk/gb/newGbInfo?hcno=22EA6D162E4110E752259661E1A0D0A8)

✳️ **满怀激情：我特别想入职咱们公司**

- 发挥主观能动性，就能想到更多更好的方法措施，写出更合适的求职简历！
- 做一次销售，达成在有限时间内获得最高薪酬的目标，就会想到更多更好的方法措施。这是很有意思的事情！

✳️ **自信满满：岗位三千，我只取一个就之！我是最棒的！**

- 咱们学校是很有竞争力的！截至2024年6月20日，全国高等学校共计3117所，其中：普通高等学校2868所，含本科学校1308所、高职（专科）学校1560所；成人高等学校249所。本名单未包含港澳台地区高等学校。更多详情可参考：[全国高等学校名单🔗](http://www.moe.gov.cn/jyb_xxgk/s5743/s5744/A03/202406/t20240621_1136990.html)。
- 咱们学校是很有竞争力的！第二轮“双一流”建设高校及建设学科名单中，咱们是 140 多所高校之一。更多详情可参考：[第二轮“双一流”建设高校及建设学科名单🔗](http://www.moe.gov.cn/srcsite/A22/s7065/202202/t20220211_598710.html)。
- 参考某大模型估计和应届毕业生数量，每年大致有千万级岗位提供给应届毕业生。IT/互联网/游戏行业招计算机类岗位约 200-300万，加上其他招聘计算机类毕业生的行业（如电子/通信/半导体行业、金融行业等），则更多。
- 某大厂的销售岗宣讲会，已连续几年在学校举办。说明该大厂认可毕业生质量，学校和学院都已经多名毕业生入职该大厂的销售岗和技术岗。
- 咨询某大厂某产品线HR，得到答复：该产品线研发岗也招咱们学校的。
- 手握数十个 offer 的牛娃，也只能入职一个单位。牛娃最终选择后空出来的岗位，也是机会！

✳️ **怎么好怎么写**

- 只要让用人单位觉得“你很厉害！你很适合所投岗位！”，怎么写都是可以的！

## 简历内容

应届毕业生的求职简历，一般应按如下顺序呈现如下相关信息。当然，建议的顺序也不是绝对的，但一般应让用人单位一眼扫过去就能看到想知道的信息。或者你希望用人单位优先看到的信息，比如在某大厂有相关的实习，于是可以将 [实习经历] 放在 [项目经历] 的前面。

<!-- 👌👍🆖 ✅ℹ️✍️ -->

👤 **基本信息**

- 照片：积极向上、青春阳光的形象照。
- 姓名，手机，邮箱，等。
- 其他有助于获得 offer 的信息。

🎓 **教育背景**

- 学校，专业，入学毕业时间。
- GPA：排名靠前则呈现。
- 核心课程，或和求职岗位强相关课程。一行左右即可。研究生求职则不写。
- 荣誉奖项，等。
- 论文专利，等。
- 其他有助于获得 offer 的信息。

🎯 **专业技能**

- 参考岗位要求，描述你具备的相关技能。岗位要什么，就写什么。
- 以及技能的程度（精通 > 掌握 > 熟悉 > 了解）。
- 其他有助于获得 offer 的专业技能（通常不是吹拉弹唱）。

💼 **项目经历**

- 应届毕业生的项目来源很多，不仅仅是和企业做项目。
- 参考网上教程中的项目样例，从头到尾敲一遍代码，也是项目。
- 通常可参考 STAR 法则叙述。既能呈现用人单位想知道的信息，同时也能说明你很强。
- 项目经历描述的目的只有一个：我很棒！我很适合该岗位！

👣 **实习经历**

- 有相关性较强的实习经历，是最好的。
- 往届没有实习经历的校友，也找到了心仪企业的心仪岗位。
- 通常可参考 STAR 法则叙述。既能呈现用人单位想知道的信息，同时也能说明你很强。
- 实习经历描述的目的只有一个：我很棒！我很适合该岗位！

📚 **校园经历**

- 其他经历（不是项目经历，不是实习经历）写在这里。
- 有助于获得 offer 的经历，都可以写。
- 用于佐证你的优秀品质 / 软能力。
- 也应参考 STAR 法则叙述。相比项目经历/实习经历，通常要更精炼些。
- 尽量写。用人单位是关注校园经历的。

🚀 **自我评价**

- 类似总结。岗位要什么，就写什么。
- 描述偏软能力（沟通、协作等）。一般不重复专业技能。
- 3~4行。每行写 1 个软能力。
- 写少了，觉得你不重视；写多了，用人单位来不及看。

## 基本信息

建议呈现：

❇️ 名字

- 名字写大大的，不一定按模板写“姓名：张三”，以方便用人单位在一沓简历中查找。

❇️ 电话

- 可以直接写 13951234567（或者前面加个手机图标），不会有歧义的。不一定按模板写“手机：13951234567”。
- 可以在手机号后加 `（微信同号）`。
- 求职期间，建议设置微信，可以通过手机加微信。
- 求职期间，微信名字可以改得普通些。比如王小明同学的微信就叫 `王小明` 或 `小明`。

❇️ 邮箱

- 可以直接写 zhansan@abc.com（或者前面加个邮箱图标），不会有歧义的。不一定按模板写“邮箱：zhansan@abc.com”。
- 所留邮箱要保证在国内正常网络情况下可以收取邮件。
- 🆖 通常不留 gmail 邮箱。

❇️ 其他角色

- 班委/团委（如有）
- 中共党员/预备党员（如是）
- 学生会部长/干事（如是）
- 🆖 通常不写共青团员

❇️ 照片

- 微笑的正装照/形象照，能体现青春活力、良好的精神面貌的那种。
- 不是严肃的证件照，不是手机自拍的杂乱背景照，不是卡通头像。

❇️ 求职岗位

- 写上可能有利于投递某些岗位、获得 offer。
- 不一概而论。

❇️ 其他有助于获取 offer 的信息。

可以不写：

- 🆖 年龄
- 🆖 学校。因为在 [教育背景] 段落中有了。
- 🆖 学历。因为在 [教育背景] 段落中有了。
- 🆖 身高
- 🆖 居住地
- 🆖 籍贯
- 🆖 民族
- 🆖 **期望薪资。**自己知道就好，不在简历上告诉用人单位。
- 🆖 **期望工作城市。**自己知道就好，不在简历上告诉用人单位。

**重要说明：**

- 简历上出现一些不大相关的信息，通常是简历模板导致的。一般在招聘网站上可以设置，虽然网站有一些必填信息（比如期望薪酬等），但可以不显示在电子版简历上。
- **可以不写** 的信息，是指通常对于获取 offer 没有帮助的信息，但也不一概而论。比如某企业某岗位优先要招籍贯是当地的同学，因此可以考虑呈现籍贯。
- 觉得年龄和民族，有助于获取 offer，也可以呈现。比如某同学大四毕业时才满 18 周岁，可以尝试呈现年龄，在条件相当的应聘者中可能会脱颖而出。

## 教育背景

建议呈现：

❇️ 学校

- 入学毕业时间都写年月，不写“xx年xx月~至今”。让用人单位一眼看出是否为应届毕业生。
- 可在学校名称后增加 `（211/双一流）`，呈现给用人单位知晓。

❇️ 课程

- 写主要的、和求职岗位相关的几门即可，控制在一行。
- 如果某些核心/主要课程满绩，可在课程名称后备注下。体现成绩优秀。
- 🆖 硕士的求职简历，通常就不写课程了。研究生和本科阶段的课程，都不写。

❇️ GPA

- 如GPA在 3.2 及以上可写，如：GPA 3.2/4.0(专业排名前15%)。3.2只是举例，建议专业排名 < 20% 才主动呈现在简历上。
- GPA不高则不写。但要想好面试时怎么说。
- 🆖 硕士的求职简历，通常就不写 GPA 了。本科阶段的 GPA，也不写。

❇️ 奖项荣誉

- 含金量高的写上，其他可不写。否则显得不高级。
- 简洁、明了。比如“ACM ICPC（国家大赛）三等奖（2023）”，可能比 “ACM国际大学生程序设计竞赛 Java 组中国区三等奖”好一些。
- 简洁、明了。比如“蓝桥杯（工信部主办）二等奖（2021）”，可能比“2021届 蓝桥杯 Java组 A组 省赛二等奖”好一些。
- 简洁、明了。比如 “校三等奖学金（3次）”，可能比 “xx大学 2020年综合杰出优秀奖学金A组三等奖金”、“xx 2021年 xx”、“xx 2022年 xx”要好些。
- 🆖 硕士的求职简历，不要把奖项荣誉分列成硕士和本科阶段。研究生和本科阶段的荣誉奖项，混在一起写就好。

❇️ 论文专利

- 含金量高的写上，其他可不写。否则显得不高级。
- 建议也是简洁明了。让人觉得你还厉害。

❇️ 其他信息。 其他可帮助获得 offer 的信息，也可以呈现。

## 专业技能

1. 列出和求职岗位相关的技能。 

    - **要什么写什么。** 而不是有什么写什么。
    - 不相关或不重要的，建议不写。
    - 建议不再对专业技能分类，比如这是编程语言，那是办公软件，等。一个一个写下来就好，减少层次。

2. 呈现合理的技能程度。 

    - 一般是：精通 > 掌握(即熟练掌握) > 熟悉 > 了解。
    - 慎写精通。因为需要时间积累。
    - 不夸大，更不能谦虚。
    - 确保达到岗位要求的程度。比如岗位要求掌握，自身只是熟悉，可以先写掌握，并尽快学习提升到掌握（至少是半步掌握）。否则可能过不了简历筛选关。

3. 一行写一个（或多个相关）技能。

    - 比如一行可以写：“掌握 C/C++/Python，……”，都是说编程语言。（不是说一定要把几门编程语言写一行）
    - 不要一行写：掌握 C/C++，熟悉 MySQL，了解 TCP/IP。
    - 不能只写 “掌握 Python。”，要写成：掌握 Python，了解xx框架和xx、xx等模块，能实现xx。
    - 建议不出现 `基础`、`常见`、`简单`、`入门` 等字样。

4. 重要的 + 相关的技能，放前面。
    
    - 岗位要求的第1个技能是掌握 Java 语言，则第1行就描述 Java 技能。
    - 英语四六级 & 普通话，可写一行，都是语言。但要靠后放，作为最后几项技能之一。除非岗位对英语有特别要求，则可以靠前放。
    - Office 可写，今后工作中也常用。也要靠后放。如果放在前面，会被误解计算机技能不足的。

## 项目经历

1. 来源较多，都可以作为项目写。 

    - 来自课程设计、竞赛项目、自己做的项目、毕设，等。
    
2. 说清楚是什么 / 做了什么 / 效果如何等，突出自己很厉害。 

    - 可参考 **S.T.A.R.** 法则描述。
    - 🆖 不要记流水账。不要这样描述：“xx年xx月，我和同学做了个xx项目，最终实现了要求的功能。”
    - 🆖 不要做项目科普。不要这样描述：“基于xx的方法是利用xx和xx，对现有的xx做算法改进和效率提升。xx可应用到众多领域，比如xx领域，可实现xx；比如在xx领域，可实现xx。我主要做了算法改进工作，并提升效率x%。”
    - 项目很有价值，有很多难点，我解决了很多难点，效果很棒。因此我很厉害！
    - S 和 T 可以写在一起，比如 2 行左右。A 和 R 可以写在一起，比如 2 ~ 3 行左右。既涵盖 STAR 的要点，读起来也比较自然连贯通顺（而不是 S / T / A / R 各写成1点，每点 1 ~ 2 行）。
    - 就项目论项目的结果。`获得xx比赛一等奖` 不应该是项目的结果，至少不是全部的结果。如果不确定项目结果是什么，至少可以写成 `将提升 x % / 将解决 x % 的 xxx`。

3. 符合信息安全和商业保密。 

    - 未经授权披露的信息，不要写。
    - 比如在 A 公司针对 B 客户做了 C 项目，而用人单位 D 公司和 A 公司是商业竞争关系，就泄密了。
    - 和不是非常著名的公司做了项目，可列为 `xx科技公司`，并以安全保密为理由，不透露公司名称，也是可行的做法之一。

4. 项目标题那行中，突出项目的公司、或项目名、或自己的角色。

    - 公司很有名，就突出公司。当然，要符合信息安全和商业机密，至少可以写某某世界500强企业。
    - 项目看上去很厉害，就突出项目。比如基于xx的人工智能xx项目。
    - 如果角色厉害，就突出角色。比如总体负责 / 核心成员 / 全栈工程师，等。
    - 项目的起止时间，通常写在项目标题那行的行末，而不是行首。
    - 🆖 `xx大创项目` 等，不是项目的名称。
    - 项目名称力争高大上一些。比如写成 `基于 xxx 的学生信息智能管理系统`，而不是 `学生信息数据库管理系统`。

## 实习经历

和项目经历类似的写法和要求。按投递企业和岗位不同，可考虑放在 [项目经历] 之前，或之后。

## 校园经历

**校园经历可描述在校期间参加的社团活动、社会实践等。** 即不属于项目经历的、不属于实习经历的，但有助于获得 offer 的经历，都可以放在这里。

**校园经历可作为若干优秀品质 / 软能力的佐证。** 因此也建议按照项目经历的 **S.T.A.R.** 法则叙述方式，突出自己很优秀。和项目经历、实习经历相比，描述可以更精炼些。不要记流水账，不要只是写某年某月我做了啥，要进一步挖掘和呈现出优秀品质 / 软能力。

**校园经历要尽可能呈现在简历上。** 投编程等岗位的某些同学，觉得写这些有啥没啥的经历，没用！其实不然！专业技能符合岗位要求，并且优秀品质 / 软能力也符合岗位要求，则获得 offer 的几率才比较大。用人单位很看重优秀品质 / 软能力的。


## 自我评价

❇️ 可以呈现：

- **要什么写什么。** 而不是有什么写什么。
- 3~4点，每点描述 1 个软能力，或几个相关软能力。
- 每点不超过 1 行为宜。每行前面加数字或点，方便阅读。
- 软能力大多为：沟通、协作、逻辑、抗压、……。
- 可以简要呈现（重复）相关经历，以佐证的确具备某软能力。

🆖 建议不写：

- 遵纪守法。这是必须满足的底线，不必重复。
- 技术能力。这里主要是软能力。
- 吹拉弹唱。除非作为某种软能力的佐证。
- 运动健身。除非作为某种软能力的佐证。
